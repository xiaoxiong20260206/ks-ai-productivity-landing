## 2.2 组团干活：遇到大任务，主动拉人一起上

单个 Agent 再强，也有容量上限。真正的大型复杂任务，需要协同。

MyFlicker 具备了四个核心的协同能力：感知其他 Agent 的存在与状态、主动创建新的 Agent Thread、Agent 之间互相发送消息传递结果、**自组织**根据任务复杂度决定是否拆分和如何分工。

面对大任务，主 Agent 自动拆分工作，召唤多个子 Agent 并行执行，各自推进、互相协调——这就是真正意义上的 **Agent Teams**。

| 场景 | 过去 | 现在 |
|------|------|------|
| 大型复杂任务 | 只能一步步串行，AI 容易跑偏 | 主 Agent 自动拆分，多子 Agent 并行协同 |
| 跨领域任务 | 需要手动切换不同工具 | Agent Teams 各专其职，协同完成 |
| 超出单 Agent 容量 | 无解，需要人工分批处理 | 自动扩容，组队作战 |

# 03 进阶能力：让 AI 真正为你定制

## 3.1 SubAgent：给 AI 配置你的专业助手

不同场景需要不同专长。SubAgent 让你可以创建专属的子 Agent，面向特定场景持续优化。

**典型例子：创建一个代码审查专家**

```
/subagent-manager 帮我创建一个 code-review agent，代码审查专家，
每次写完代码或修改代码后立即调用，从质量、安全、可维护性多角度分析
```

创建后，每次有代码变更，专属的 code-review agent 就会自动介入检查。记忆功能配合 SubAgent，还能让 AI 自动学习你的偏好，主动调用合适的 Agent。

## 3.2 Hooks：在关键节点插入你的业务规则

在任务执行的关键节点插入自定义逻辑，让 AI 按你的规则行事。

**典型例子：安全删除（移动到回收站而非直接删除）**

```
/hooks-manager 帮我创建一个 hooks，在执行 rm 或者 rm -rf 时，
改为移动到回收站
```

Hooks 让 AI 的执行更安全、更可控，适合有明确业务规则约束的场景。

## 3.3 定时任务：让 AI 真正变成自动化流水线

设定周期性任务，AI 自动在指定时间执行，完成后通过 KIM 通知你结果。

**典型例子：每天自动拉取 Team 任务，发送 KIM 提醒**

```
从 team 平台 https://team.corp.kuaishou.com/ 看看我有哪些任务，
并通过 KIM 发给我
```

定时任务 + Cloud Agent，让 AI 真正成为你的自动化流水线，而不只是问答工具。
